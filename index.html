Quickstart

Two commands. Works in every project. Claude Code gets all 34 buildlog tools immediately.

$ uv tool install buildlog # or: pipx install buildlog
$ buildlog init-mcp --global -y # register MCP + agent instructions
$ buildlog mcp-test # verify: 34 tools registered

That's it. buildlog is now ambient. It captures, extracts, and learns across every session in every repo.

Per-project setup

$ uv pip install buildlog
$ buildlog init --defaults # scaffold buildlog/, register MCP, update CLAUDE.md

With extras

$ uv pip install "buildlog[all]" # embeddings + LLM extraction + qortex backend
# or pick: [embeddings] [openai] [anthropic] [ollama] [qortex]

Python 3.10+. The qortex learning backend requires 3.11+.

How it works

buildlog runs a closed feedback loop. Every component exists because the previous one wasn't enough.

Every stage is independently useful. You can capture without extracting, extract without the bandit, or render without experiments. But the full loop is where compound improvement happens.

The buildlog stack

Hover to flip. Eight components, each independently useful.

MCP Server
hover to flip
MCP Server

34 tools via Model Context Protocol. Your agent calls buildlog during sessions: log commits, run reviews, query bandit state, close feedback loops. Native to Claude Code.

Thompson Sampling
hover to flip
Thompson Sampling

Beta-Bernoulli contextual bandit. Error classes partition state space. Seeds start at Beta(3,1), non-seeds at Beta(1,1). ~20 observations for meaningful differentiation.

Extraction
hover to flip
Extraction Pipeline

Regex (fast, free), OpenAI, Anthropic, or Ollama extractors. Semantic dedup via sentence-transformers or OpenAI embeddings. Pluggable at every stage.

Review Gauntlet
hover to flip
Review Gauntlet

Multi-persona code review. Security, architecture, testing, performance. Runs on commits, categorizes by severity, files GitHub issues. Configurable stop conditions.

Experiments
hover to flip
Experiment Framework

Start/stop sessions. Log mistakes with typed error classes. Compute Repeated Mistake Rate across sessions. Statistical evidence, not vibes.

Multi-Agent
hover to flip
Multi-Agent Render

Same knowledge base renders to Claude Code, Cursor, Copilot, Windsurf, Continue.dev, VS Code. Switch agents, keep what you learned.

Global SQLite
hover to flip
Global SQLite

Single database at ~/.buildlog/buildlog.db. WAL mode. Project isolation via SHA-256 hashed IDs. Legacy JSONL fallback for older installs.

Pluggable Backends
hover to flip
Pluggable Backends

Builtin bandit or qortex for cross-consumer credit propagation. LearningBackend protocol with two shipped adapters. Swap without data loss.

Current limits

v0.15. The full loop works. These are the open problems.

Extraction quality

Regex misses nuance. LLM extractors cost money. The middle ground is still being found.

Feedback signals

RMR works but needs manual tagging. Automatic signals (test results, revision distance) are on the roadmap.

Credit assignment

Multiple active rules, one outcome. The system doesn't yet isolate which rule was responsible.

Single-agent

Shared learning across agents in a session is designed but not shipped. Rendering to multiple agents works.

See the roadmap: cross-project convergence, emergent rule graphs, contextual bandits with embedding-space context vectors.

FAQ

How is this different from editing CLAUDE.md by hand?

Manual editing is step 1. buildlog is steps 2 through 5: extracting patterns from real work instead of guessing, selecting which rules to include based on outcomes instead of intuition, measuring whether they reduce mistakes, and demoting the ones that don't. You can edit CLAUDE.md forever and never know what's working.

Does this need an API key?

No. The base install uses regex extraction and local computation. LLM-backed extraction (OpenAI, Anthropic, Ollama) is available via extras for richer patterns. The core loop runs without external calls.

How many observations before the bandit differentiates?

Seeds start at Beta(3,1). With ~20 observations per context, the posteriors separate. The system is deliberately conservative with limited data to avoid killing rules prematurely.

What data leaves my machine?

Nothing, by default. Everything is in ~/.buildlog/buildlog.db (local SQLite). If you enable LLM extraction via API key, prompts go to the provider you chose. Bandit, experiments, and the core loop are entirely local.

Can I use this without Claude Code?

Yes. The MCP server is Claude Code-native, but rule rendering works for Cursor, Copilot, Windsurf, Continue.dev, and VS Code. The CLI works standalone: buildlog commit, buildlog experiment start, buildlog promote --target cursor.

Is this production-ready?

Used in production daily across multiple projects. 1,254 tests. Stable API surface. But extraction quality is uneven (see limits above) and credit assignment is an open problem. Use it, measure it, report what breaks.
