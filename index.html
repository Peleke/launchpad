The ship / iterate / kill decision framework

This is from Week 4. It’s the real thing — not a teaser. Genuine standalone value, because we believe giving away 80% earns trust for the other 20%.

The three questions:

  1. Does it work? (Quality) — Run your behavioral test suite. Compare agent output quality to human baseline. Measure accuracy, completeness, hallucination rate. Pass rate >85% with comparable quality = ship candidate. Below 70% on critical tasks = kill candidate.
  2. Is it worth the cost? (Economics) — Cost per task: LLM API + infrastructure + engineering maintenance. Compare to human cost per task. Don’t forget error correction cost. If the agent is wrong 30% of the time and a human has to fix it, add that.
  3. Can you operate it? (Operational Readiness) — Do you have traces for every agent decision? Can you detect quality regression automatically? Have you threat-modeled the tool access? Do you have on-call runbooks? Score 5/5: ship. Score <3: you’re not ready.

If you land in KILL: that’s not a failure. That’s engineering. You saved your team months of investment in a system that wouldn’t have worked. Document why, share the analysis, and redirect resources.
